## 一、架构与端口速览

```
┌─────────────────┐    /api/*    ┌──────────────────────┐
│  前端 Vue3       │ ───────────▶ │  后端 SpringBoot      │
│  :5173           │             │  :8080                │
```

## 四、第一步：准备数据库（MySQL）

### 1. 启动 MySQL

- **Windows 服务方式**：
  ```powershell
  net start MySQL80
  ```
  > 服务名可能是 `MySQL`（不带 80），在「服务」里确认实际名称。
- **XAMPP / WAMP**：直接点启动 MySQL 即可。

### 2. 创建数据库并导入表结构

本项目数据库名为 `book_cycle`，表结构拆在 `database/` 下的 7 个文件里。一次性导入：

```powershell
# 假设本机 MySQL root 密码为 0428（与 application.yml 默认一致）
mysql -uroot -p0428 -e "CREATE DATABASE IF NOT EXISTS book_cycle CHARACTER SET utf8mb4;"
Get-ChildItem database\book_cycle_*.sql | ForEach-Object {
    mysql -uroot -p0428 book_cycle < $_.FullName
}
```

导入成功后验证：

```powershell
mysql -uroot -p0428 book_cycle -e "SHOW TABLES;"
```

应看到 `address / book / cart / category / order_info / user / user_behavior` 七张表。

> ⚠️ **密码不一致怎么办**：如果本机 MySQL 密码不是 `0428`，后面启动后端 / AI 服务时用环境变量覆盖（见下两步），不必改 `application.yml` / `.env`。

---

## 五、第二步：启动后端（SpringBoot :8080）

打开 VS Code 终端（建议新建一个页签），执行：

```powershell
cd backend
mvn spring-boot:run
```

等待控制台出现：

```
Started BackendApplication in X.XXX seconds
```

即启动成功，后端地址 `http://localhost:8080`。

**若 MySQL 密码不是 0428**，先设环境变量再启动：

```powershell
$env:BOOK_DB_PASSWORD = "你的密码"
mvn spring-boot:run
```

> JWT 秘钥在 `application.yml` 里有本地开发默认值，直接跑即可；**生产环境**务必用 `JWT_SECRET` 环境变量注入高强度随机值。

## 六、第三步：启动 AI 微服务（FastAPI :8000）

> 这一步提供「自然语言找书 / 智能推荐 / 库存查询」的 AI 能力，依赖大模型 API Key。
> **Key 已配好**（DeepSeek + 智谱 GLM），详情见 `ai-service/README.md` 与 `ai-service/.env`。

新建一个终端页签：

```powershell
cd ai-service

# 1. 建虚拟环境（隔离依赖，推荐）
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动（--reload 支持热更新）
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

等待出现：

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Loaded XX books from MySQL
```

表示已连上数据库并加载书籍语料，AI 服务就绪。

### 没有 Key 也能跑（降级模式）

即使不填任何 Key，服务也会启动，只是大模型生成能力关闭，自动降级为「纯检索 + 规则回复」。验证：

```powershell
curl http://localhost:8000/health
```

返回 `llm_enabled: false` 即降级模式，浏览器仍可联调前端与检索。

### 多模型自动切换（fallback）

`.env` 里配了 DeepSeek（主）+ 智谱 GLM（备用）。主模型额度耗尽 / 失败时**自动切下一个**，前端角标会显示实际命中的平台。当前真实情况：DeepSeek 余额不足 → 自动用免费的智谱 GLM 兜底。

1. 打开 `http://localhost:5173/`，右下角有 **🤖 悬浮按钮**。
2. 点击展开「AI 找书助手」。
3. 输入：

## 九、常见问题排查

### 1. 端口被占用（Address already in use / 10048）

```powershell
# 查占用端口的 PID
Get-NetTCPConnection -LocalPort 8000      # AI 服务
Get-NetTCPConnection -LocalPort 8080      # 后端
Get-NetTCPConnection -LocalPort 5173      # 前端

# 结束进程
Stop-Process -Id <PID> -Force
```

> 进程杀掉后端口可能短暂处于 `TIME_WAIT`（约 1 分钟），属正常，稍等或换端口即可。

### 2. 前端 AI 对话 404（路径 `/ai/api/ai/chat`）

**根因**：前端 axios 的 `baseURL` 是 `/ai`，请求实际路径已是 `/ai/api/ai/chat`；Vite 的 `/ai` 代理必须**剥掉前缀**再转发，否则 AI 服务收到多一层 `/ai` 而 404。
**修复**（已在 `vite.config.js` 中）：

```js
'/ai': {
  target: 'http://localhost:8000',
  changeOrigin: true,
  rewrite: (p) => p.replace(/^\/ai/, '')   // 剥掉 /ai 前缀
}
```

改完**必须重启 `npm run dev`** 才生效。

### 3. 后端编译失败 / Maven 找不到

- 确认已安装 Maven 且 `mvn -v` 有输出（项目无 `mvnw` wrapper）。
- 首次 `mvn spring-boot:run` 会下载大量依赖，耐心等待。

### 4. AI 服务连不上数据库

- 确认 MySQL 已启动、`book_cycle` 库存在。
- 密码不一致时设环境变量：`$env:BOOK_DB_PASSWORD="你的密码"` 后再启动 uvicorn。

### 5. 前端 `npm run build` 报错找不到 terser

已在 `package.json` 加入 `terser` 依赖，正常 `npm install` 后不再报错。

## 十、已做的代码质量加固（团队须知）

作为资深开发介入时，已修复以下隐患，团队成员后续改动请保持：

1. **SQL 注入修复**：`BookController.list` 的 `keyword` 改为 `?` 参数化查询，杜绝拼接注入。
2. **密钥外部化**：DB 密码 / JWT 秘钥全部走环境变量（`application.yml` / `ai-service/.env`），禁止明文入库；`.env` 已被 `.gitignore` 忽略。
3. **生产禁用堆栈泄露**：`application.yml` 中 `include-stacktrace/include-exception` 已关闭。
4. **死代码接上**：`recommendByItemCF`（物品协同过滤）新增端点 `/api/recommend/item-cf/{bookId}`。
5. **补测试**：`RecommendServiceImplTest`（Mockito，无需 DB），树立测试文化。
6. **AI 能力落地**：FastAPI 微服务 + BM25 检索 + 多模型 fallback + 前端对话窗，简历「AI应用开发」方向的可演示作品。
